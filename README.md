# GhostPortal

GhostPortal is a production-quality internal employee portal built with Django 5.x/6.x for use in CTF (Capture The Flag) challenges. It features secure session authentication, custom security hardening, dynamic Bootstrap 5 styling with a corporate aesthetic, and a password-protected handbook file.

## Stack

* **Framework:** Django 5.x / 6.x
* **Database:** SQLite
* **Styling:** Bootstrap 5 (Dark navy, white, and light gray corporate theme)
* **Configuration:** `python-decouple` (loaded via `.env`)
* **Security:** CSRF protection enabled, admin panel completely removed to prevent exposure, and custom 404/500 handlers.

---

## Features

1. **Landing Page (`/`):** Styled login portal interface with company logo/identity.
2. **Login Page (`/login/`):** Secure form utilizing Django's standard session authentication. Includes automatic pre-provisioning of `guest` and `admin` accounts:
   - **Guest credentials:**
     - **Username:** `guest`
     - **Password:** `guest123`
   - **Admin credentials:**
     - **Username:** `admin`
     - **Password:** Autogenerated secure random string stored as a hashed entry (non-discoverable).
3. **Guest Dashboard (`/dashboard/`):** Displays a welcoming dashboard with a downloads section.
4. **Hidden Hint:** Includes a hidden source comment on the dashboard page pointing to the password for the document:
   `<!-- The document password isn't random. Think about who you are. -->`
5. **Secure PDF Download:** Serves a password-protected PDF (`Employee_Handbook.pdf`) requiring authenticated access. The PDF password is `guest123`.

## Installation & Setup

1. **Clone or navigate to the project directory:**
   ```bash
   cd Sphostication
   ```

2. **Set up a virtual environment and activate it:**
   ```bash
   python -m venv venv
   # On Windows:
   venv\Scripts\activate
   # On macOS/Linux:
   source venv/bin/activate
   ```

3. **Install the dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

4. **Set up the environment variables:**
   Copy the example environment configuration:
   ```bash
   cp .env.example .env
   ```
   *Note: On Windows, use `copy .env.example .env`.*

5. **Perform database migrations:**
   This applies the default schema and automatically provisions the `guest` and `admin` users.
   ```bash
   python manage.py migrate
   ```

6. **Run the local development server:**
   ```bash
   python manage.py runserver
   ```
   Access the portal locally at `http://127.0.0.1:8000/`.
